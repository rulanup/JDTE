# JDT Extras

JDT Extras (`jdte`) 是 [Just Dire Things](https://www.curseforge.com/minecraft/mc-mods/just-dire-things) 的 NeoForge 扩展模组，为 JDT 增加升级卡、扩展机器、时间加速、范围控制和自动化设备。

当前版本：`0.5.5`

[English README](README_EN.md)

## 主要功能

### 升级系统

普通机器拥有 4 个升级槽，扩展机器拥有 8 个升级槽。空升级槽会显示当前机器支持的升级类型、已安装数量和上限；手持升级卡蹲下右键机器可以直接插入，FTB Ultimine 可对选区机器批量插入。

| 升级 | 主要效果 | 上限 |
|---|---|---:|
| 容量 | FE 和流体容量按 2 的幂提升 | 3 |
| 超频 | 提高运行速度和传输批量，增加能耗 | 1 |
| 降频 | 降低运行速度和能耗 | 1 |
| 流体 | 仅提升流体容量 | 3 |
| 存储流体 | 为 JDT 点击器增加内部储罐 | 1 |
| 发电机 | 以更多燃料换取更高发电量 | 1 |
| 范围 | 提高范围半径和偏移上限 | 2 |
| 过滤 | 每张增加 9 个过滤槽 | 2 |
| 创造 | 免除 FE；包含超频效果 | 1 |
| 时运 | 凝胶发生器、水晶培育机和温室大棚；每张增加 1 级原版时运 | 8（温室 3） |
| 精准 | 水晶培育机专用；按原版精准采集逻辑回收水晶，与时运互斥 | 1 |
| 抢夺 | 生物粉碎机和战利品制造机专用 | 6 |
| 锋利 | 生物粉碎机专用；每张增加 5 点伤害 | 6 |

超频和降频不能同时安装。创造升级同时具备超频升级的相关功能。

### 时间与扩展机器

- 初级时间加速器：默认 16x，超频或创造升级后 32x，只消耗 JDT 时间流体。
- 高级时间加速器：可调 1-64x，超频或创造升级后 128x，消耗时间流体和 FE，时间流体成本为初级版的 2 倍。
- 扩展高级时间加速器：8 槽变体，可调 1-512x，超频或创造升级后 1024x，时间流体成本为初级版的 5 倍。
- 三档时间加速器共用统一调度器；重叠倍率完整叠加，通过区块目标发现、已付费虚拟 tick 队列和动态 MSPT 余量降低大范围与多机器开销，并支持 AE2 `IGridTickable` 设备。
- 扩展升级可将 JDT T2 点击器、方块破坏器、方块放置器、方块替换器、投掷器、传感器、流体收集器和流体放置器转换为 8 槽版本，并保留机器数据。

### 自动化机器

| 机器 | 功能 |
|---|---|
| 高级物品拾取器 | 在掉落物进入世界前写入朝向一侧的容器；支持超大堆叠预转移、AE2 `ME_STORAGE` 和 ExtendedAE 接口 |
| 实体抑制器 | 抑制实体更新、禁止实体生成、禁用实体渲染、禁用方块实体渲染或禁用粒子 |
| 范围屏蔽器 | 围困范围内生物，或阻止玩家磁铁吸取范围内掉落物 |
| 粘胶激活器 | 自动执行 JDT 粘胶相关操作 |
| 凝胶发生器 | 执行 JDT 凝胶扩散转换；时运升级可提高 JDT 原矿产量 |
| 流体稳定器 | 在配置范围内执行 JDT FluidDrop 转换 |
| 物品/流体发送器 | 将内部物品或流体发送到范围目标 |
| 物品/流体接收器 | 从范围目标拉取物品或流体 |
| 水晶培育机 | 消耗时间流体和 FE，以可调 1-512x 或超频 1024x 催生通用母岩，自动输出成熟晶簇并支持时运或精准采集 |
| 温室大棚 | 4 个可堆叠植物模板，以原 512 工作量为 1x，可调至 32x，超频/创造升级强制 64x；支持作物、花朵、树苗和模组植物、四向连接外观、时运、JEI，并将新产物直接生成到相邻容器 |
| 生物工厂 | 使用可复用刷怪蛋或资源蜜蜂笼、食物/授粉物、FE 与四类流体生产物品或流体；可调 1-32x、超频/创造 64x，默认 2x4 八格输出，支持自动 I/O、资源蜜蜂 JEI 配方和四档产量升级 |
| 生命繁育器 | 在设定范围内使用 2x2 饲料自动配对标准动物或村民，以 1-32x 推进幼体成长与繁殖冷却；超频/创造可直接结算，支持刷怪蛋黑白名单过滤、按繁育时间计费的生命流体、4x2 掉落物回收、范围、红石与自动 I/O |
| 工厂打包机 | 将方块、含物品方块实体、非玩家实体和计划 Tick 事务化迁移到 UUID 包裹；支持运行中源区域重捕获、Mekanism 反应堆安全停机与放射性传输器无泄漏拆除、多方块从属结构联动拆解、结构预览、旋转、AE2 移动策略、异步 I/O、回滚与重启续作 |
| 生物粉碎机 | 使用假玩家击杀目标并产生战利品和经验流体；支持刷怪笼和专用升级 |
| 生命提取器 | 将目标生命值转换为生命流体，不产生普通掉落和经验 |
| 灌注机 | 执行凝胶、物品及动态刷怪蛋灌注配方 |
| 高级炼药机 | 六步有序酿造，支持配方锁定、水和时间流体、自动 I/O、独立烈焰粉外部输入开关及 JEI 酿造链 |
| 战利品制造机 | 使用刷怪蛋模板、生命流体、时间流体和 FE 制造生物战利品 |

### 自动输入输出

具备真实物品或流体接口的机器可以按世界绝对方向配置：

`关闭 -> 自动输入输出 -> 自动输入（橙色） -> 自动输出（蓝色） -> 关闭`

机器不支持的模式会自动跳过。发送器只提供自动输入，接收器只提供自动输出。自动 I/O 默认批量为每次 10,000 件物品或 1,000,000 mB；发送器和接收器默认批量为 64 件或 20,000 mB，安装超频或创造升级后提高传输能力。

### 蚀空合金扳手

- 右键旋转兼容机器，蹲下右键可保留 NBT 搬运支持的机器。
- 注册到通用扳手标签，可使用 AE2 等模组的原生旋转和拆除逻辑。
- 两次左键选择角点并显示 JDT 风格范围预览和尺寸；左键范围机器可写入选择。
- 选择写入后仍保持锁定，可复用到多台机器；Shift+左键清空。
- 支持 FTB Ultimine 批量操作，创造模式持有时不会误破坏方块。

### 兼容与信息显示

- Jade 显示机器已安装升级的图标、名称和合并数量。
- JEI 提供凝胶发生器、灌注机、高级炼药机和战利品制造机分类。
- 可选兼容 FTB Ultimine、AE2/ExtendedAE、Mekanism、Apothic Spawners 和 Draconic Evolution。

## 安装要求

- Minecraft `1.21.1`
- NeoForge `21.1.233+`
- Just Dire Things `1.5.7+`
- Java `21`

将 `jdte-x.x.x.jar` 同时放入客户端和服务器的 `mods` 文件夹。

## 开发构建

```bash
./gradlew compileJava
./gradlew jar
./gradlew runClient
./gradlew runServer
```

详细架构与开发流程参见 [开发文档.md](开发文档.md) 和 [AGENTS.md](AGENTS.md)。版本变化参见 [CHANGELOG.md](CHANGELOG.md)。
