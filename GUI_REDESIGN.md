# JDT Extras — GUI 重新设计方案

## 概述

目前所有 JDTE 机器都使用 JDT 的 `BaseMachineScreen`，固定面板宽度 176px，额外宽度 `extraWidth=60`（总计 236px）。升级槽和过滤槽以**浮动弹窗**形式渲染，可以在屏幕上任意拖动。本文档分析每台机器的组件，识别重叠区域，并提出嵌入式（无弹窗）GUI 布局方案。

## 2. 当前 GUI 布局分析

### 2.1 基础尺寸

所有 JDTE 机器继承 JDT 的 `BaseMachineScreen`，配置如下：
- **基础面板**：176 × `imageHeight`（可变）
- **额外宽度**：60px（右测流体条区域）
- **总面板宽度**：236px
- **`imageHeight`**：根据机器槽数量和玩家背包自动计算
- **`getGuiLeft()`**：`(screenWidth - (176 + 60)) / 2`
- **`getGuiTop()`**：`(screenHeight - imageHeight) / 2`

| 问题 | 涉及组件 | 描述 |

| **O1: 升级弹窗悬浮** | K, L, C/D | 升级弹窗是浮动窗口，与主面板右测的流体条 (C/D) 重叠。切换按钮 (L) 在 `guiLeft+184`，位于 extraWidth 区域内，可能遮挡流体条。 |

## 3. 嵌入式 GUI 方案（无浮动弹窗）

### 3.1 设计原则

1. **所有组件嵌入**单个固定尺寸面板 — 无浮动弹窗
2. **跨机器族布局一致**
3. **间距充足** — 无重叠
4. **升级槽始终可见** — 无隐藏功能
5. **过滤翻页在面板内** — 无按钮超出边界
6. **玩家背包**保持在底部相同位置

### 3.2 所需槽位/组件清单

针对**最复杂的机器**（扩展生物粉碎机 + 所有升级）：

| 组件 | 数量 | 最小宽度 | 最小高度 |
|------|------|---------|---------|
| 升级槽（扩展） | 8 | 4×18=72（4列） | 2×18=36（2行） |
| 过滤槽（每页） | 9 | 9×18=162 | 1×18=18 |
| 机器：物品输出 | 16 | 4×18=72（4列） | 4×18=72（4行） |
| 机器：抢夺槽 | 2 | 2×18=36 | 1×18=18 |
| 机器：锋利槽 | 2 | 2×18=36 | 1×18=18 |
| 能量条 | 1 | 18 | ~64 |
| 流体条 | 1 | 18 | ~64 |
| 进度箭头 | 1 | 24 | 16 |
| 模式按钮 | 1 | 16 | 16 |
| 玩家背包 | 27+9 | 9×18=162 | 6×18+4=112 |

凝胶发生器特有：

| 组件 | 数量 | 最小宽度 | 最小高度 |
|------|------|---------|---------|
| GEL + FOOD 槽 | 2 | 1×18+填充 | 2×18=36 |
| 输入槽 | 4 | 1×18+填充 | 4×18=72 |
| 输出槽 | 4 | 1×18+填充 | 4×18=72 |
| 自动平衡按钮 | 1 | 60 | 18 |

### 3.3 计算的最小 GUI 尺寸

玩家背包宽度是**主要宽度约束**：9 槽 × 18px = 162px，加上左侧能量条（18px）和右侧流体条（18px），再加填充。

**最小宽度 = 18（能量）+ 2（填充）+ 162（背包）+ 4（填充）+ 18（流体）+ 2（填充）= 206px**

过滤槽与背包对齐也是 9×18=162px，所以 206px 可行。

对于升级网格，有两种方案：
- **水平布局**：在背包上方添加一排升级槽
  - 8 槽 × 18px + 间距 = 8 × 18 = 144px — 可在 162px 内放下
- **网格布局**：4×2 网格 = 72px 宽，增加 36px 高度

**建议的统一 GUI 尺寸：**

```
面板宽度：176 + 60 = 236px（保持不变）
面板高度：按机器族变化
```

核心问题是**高度**——需要将所有组件垂直排列：

对于生物粉碎机（最复杂）：
- 模式按钮 + 进度条 + 流体条
- 输出槽 (4×4 网格) + 抢夺/锋利槽
- 升级槽 (4×2 网格)
- 过滤槽 (1×9 行)
- 玩家背包

有两种方案：增加宽度将升级槽内联，或增加高度添加升级行。方案 2（增加高度）更清晰，因为玩家习惯纵向 GUI 布局。

## 4. 当前问题与解决方案总结

| # | 当前问题 | 建议方案 |
|---|---------|---------|
| 1 | 升级槽在浮动弹窗中（关闭时隐藏） | 将 4×2 升级网格嵌入主面板，始终可见 |
| 2 | 过滤翻页按钮超出面板边界 | 将上一页/下一页按钮移至面板内部，内联显示 |
| 3 | 模式按钮悬空在面板左侧 (x=-22) | 移入面板，例如能量条和机器区域之间 |
| 4 | 自动平衡按钮严重偏左 (x=-64) | 移入面板，进度箭头下方 |
| 5 | 生物粉碎机抢夺/锋利槽固定在右侧 (152, ...) | 保留原位，但确保不与升级区域重叠 |
| 6 | 升级槽与主上下文脱节 | 始终可见 = 更好的用户体验，无需切换按钮 |
| 7 | 生物粉碎机输出槽在 (8,18) — 太靠左，与能量条区域重叠 | 右移以容纳能量条 |

## 5. 迁移注意事项

- 移除升级弹窗意味着移除 `BaseMachineScreenMixin` 中的升级切换/位置逻辑
- 过滤翻页按钮需要重新定位到面板边界内
- 每个机器族需要自定义 `imageHeight` 覆写
- `extraWidth=60` 可以考虑标准化或减少
- 生物粉碎机的抢夺/锋利槽已在主面板中（不在弹窗中）— 保持不变
- 扩展机器（8 升级槽）vs 普通机器（4）需要不同的行数或紧凑布局
