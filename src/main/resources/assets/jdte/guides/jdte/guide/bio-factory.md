# 生物工厂

<BlockImage id="jdte:bio_factory" scale="2" />

生物工厂使用可重复利用的刷怪蛋或资源蜜蜂笼作为样本，将食物、授粉物、培养流体、FE、生命流体和时间流体转化为生物产物。烈焰合金框架内显示的生物只是客户端模型；机器不会在服务端生成、寻路或 Tick 真实实体。

## 槽位与流体

| 位置 | 用途 |
|------|------|
| 一列四格输入 | 第一格是带蜜蜂蛋底图的样本槽；下面三格放食物、授粉物、环境催化物或消耗材料 |
| 2x4 输出槽 | 默认 8 格物品产物，每张容量升级增加 8 格内部容量，最多 32 格 |
| 生命流体 | 有足够流体时默认将本轮产量提高到 2 倍 |
| 时间流体 | 有足够流体时启用速度按钮，可调 1-32x；超频或创造升级最高 64x |
| 培养流体 | 支持以流体为授粉条件的资源蜜蜂及数据配方流体输入 |
| 产物流体 | 独立输出牛奶等流体，不会被输入管道抽走 |

基础生产会消耗 FE。三个材料槽无顺序要求，每项材料独立声明是否消耗；食物、授粉物和环境方块通常可复用，碗、花、岩浆膏和紫水晶碎片等平衡材料只在成功生产后扣除。没有生命流体或时间流体时仍能以基础产量和基础速度运行；只有实际成功写入产物后才扣除资源。创造升级移除全部机器资源消耗并包含超频效果。

JDTE 内置动物配方使用基础生命/时间流体成本。Productive Bees 等动态外部配方默认使用 10 倍时间流体与 5 倍生命流体成本，这两个倍率可在服务端配置中调整。

## 生物与蜜蜂兼容

- 内置配方进一步覆盖原版蜜蜂蜜脾、哞菇牛奶、雪傀儡雪球、16 色羊毛和三种蛙明灯。蛙明灯每轮消耗 4 个岩浆膏，并分别用草方块、浮冰和沙子选择赭黄、青翠和珠光变体。
- 平衡型配方包括：山羊用石头催化并以 25% 概率消耗 4 个紫水晶碎片生成随机有效音色的山羊角；棕色蘑菇、碗和小型花生成带对应原版效果的可疑炖菜；猫按原版晨间礼物概率慢速生产；熊猫以 2.5% 概率产生黏液球。
- 整合包可以添加 `jdte:bio_factory` 数据配方。`inputs` 最多声明三项无序材料，每项的 `ingredient` 表示匹配条件，省略 `count` 或设为 0 表示可复用，大于 0 才会在成功结算后消耗；旧 `food`/`food_count` 配方仍兼容读取。
- 安装 Productive Bees 后，可直接使用装有具体蜜蜂的普通或坚固蜜蜂笼，也支持能够确定蜜蜂类型的刷怪蛋。
- 资源蜜蜂只验证自身数据定义的特定授粉物或授粉流体，不再把通用繁殖花朵误认为所有蜜蜂都能使用的生产条件。机器识别附属蜜蜂的精确 `flowerBlock`，并将 `flowerTag` 优先按方块标签解析、为空时才回退到物品标签，因此时间水晶块等附属模组授粉方块也可直接放入；JEI 会显示对应方块、具体物品或培养流体。
- 资源蜜蜂直接使用当前加载的 `advanced_beehive` 产出配方。蜜蜂自身的产量基因沿用高级蜂箱的原始增产公式，昼夜行为和天气耐受也会决定当前环境能否生产。
- Alpha、Beta、Gamma 和 Omega 产量升级合计最多安装 4 张。倍率与 Productive Bees 当前配置一致并相加计算；Omega 会像高级蜂箱一样直接产出对应蜜脾块。
- JDTE 内置配方支持最多 4 张抢夺升级。每张使用项目统一的抢夺额外掉落概率，对本轮已生成的物品产物追加一次复制判定；不会影响 Productive Bees 动态配方。
- 耐力和脾气仍完整保存在蜜蜂笼样本中。它们在 Productive Bees 中用于真实蜜蜂蜇刺、攻击和出巢行为；生物工厂不生成或 Tick 实体，因此不会虚构额外生产倍率。

JDTE 的生物工厂 JEI 分类同时显示数据驱动配方和当前数据包中实际加载的 Productive Bees 高级蜂箱配方，包括四格输入、概率产物、流体槽和 FE。悬停生产进度图标会显示配方基础 Tick 数和秒数。

## 性能

机器只在样本、三个材料槽或培养流体变化时重新解析配方。进度通常按默认 20 Tick 间隔合并结算，在高倍速即将完成时会提前结算以保持真实倍率；产物先模拟容量再统一写入，不会扫描范围、生成掉落物实体或执行生物 AI。客户端只为视距内机器缓存一个展示实体，最大显示距离为 32 格。相邻生物工厂通过原生方块状态连接玻璃墙、立柱和顶部边框，只在放置或邻居变化时更新。
