# 工厂打包机

<BlockImage id="jdte:factory_packer" scale="2" />

工厂打包机可以将设定范围内的方块、方块实体、非玩家实体和计划 Tick 剪切到便携包裹中，并在另一处恢复。机器库存、能量、流体、升级及大多数模组数据通过完整方块实体数据保存。

## 打包

1. 使用范围界面或蚀空合金扳手设置范围。打包机本身必须位于范围外。
2. 放入一个空工厂包裹并按下开始按钮。
3. 机器先扫描选区。完整框选的 Mekanism 裂变反应堆会通过公开 API 自动停机；部分框选会拒绝操作并报告反应堆完整范围。
4. 停机后重新捕获权威快照并安全写入世界存档。源方块拓扑变化会最多重新捕获 3 次，而不是立即中止。
5. 已保存实体移除后，机器再分批检查方块权限并剪切宿主方块；权限拒绝会恢复机器和实体。
6. 完成后槽内的包裹会显示尺寸、非空气方块数量和根实体数量。

## 展开

手持已填充包裹时会显示边界和经过上限裁剪的真实方块模型预览。未定位时预览跟随准星指向的方块表面；右键方块可固定摆放位置，Shift 右键可清除定位。按住范围调节键（默认左 Alt）滚动滚轮可按 90 度水平旋转。把包裹放进目标打包机后，机器会直接读取包裹的位置、尺寸和旋转。

目标范围必须完全为空。包裹文件会先被独占锁定，随后分批恢复旋转后的方块状态、方块实体、实体树、计划 Tick、光照和邻居更新；成功后包裹被消耗。未设置包裹定位时会以机器范围原点作为摆放原点。

操作期间不能取出包裹、旋转、改变范围或破坏打包机。失败的剪切与放置会分批回滚。服务器重启后，已经安全落盘的剪切或放置阶段会继续执行。

裂变反应堆必须完整位于选区内。打包机会在处理冷却剂管线、废料管线或反应堆外壳前执行停机并等待燃烧停止，避免分批剪切期间失去冷却而熔毁。为避免目标网络尚未稳定时自动启动，恢复后的反应堆保持关闭，需要玩家确认冷却与废料通路后手动启动。

内部绝对坐标链接只在原范围内重写，并同步源/目标维度；方块朝向和实体位置、朝向也随结构旋转。所有实现 JDT `AreaAffectingBE` 的机器会同步旋转 X/Z 半径与偏移。AE2 方块实体使用公开移动策略；AE2 Crystal Science 广播器会在剪切前从频段运行态下线并移除旧坐标声明，恢复后再注册新位置；物流网络节点会根据旋转后的实体位置重建附着坐标和网络注册。其他网络在完整 NBT 恢复后通过标准加载与邻居更新重建。玩家、跨维度区块票以及没有公开搬运接口且只保存在未知全局管理器中的私有状态不会被猜测修改。

黑名单方块、未加载区块和超出配置上限的范围会拒绝启动。发现黑名单方块时，聊天栏会列出本地化名称、注册 ID 和实际坐标；展开旋转后的包裹时显示的是目标坐标。为避免刷屏，最多展开 16 条并显示其余数量。拥有传送命令权限的玩家可以点击坐标传送到对应方块上方。黑名单由 `#jdte:factory_packer_blacklist` 方块标签定义，数据包可以继续扩展。

机器支持范围、容量、超频和创造升级。基础 X/Y/Z 半径默认均为 10，范围升级按 20/40 继续扩展；单轴长度和总体积的新默认上限为 128 格与 1,000,000 格。超频与创造升级将每 Tick 操作预算提高到四倍；创造升级免除 FE 消耗。默认每 Tick 最多处理 512 项，源变化最多有界重新捕获 3 次，压缩读写和预览文件读取在 IO 线程执行。
