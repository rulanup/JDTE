# 温室大棚

<BlockImage id="jdte:greenhouse" scale="2" />

温室大棚可以同时将 4 种作物、花朵或树苗作为可重复使用的种植模板，消耗时间流体和 FE 快速生成产物。透明培养舱会在客户端显示 4 株对应植物的生长过程，但不会在世界中放置、随机刻或破坏真实植物方块。

## 运行参数

| 项目 | 行为 |
|------|------|
| 1x 基准 | 每 Tick 累计 512 生长工作量 |
| 倍率 | 速度按钮可调 1-32x；超频或创造升级解锁 64x |
| 单次能耗 | 10 FE，不再随速度档位额外增长 |
| 单次流体 | 配方原成本除以 100 后向上取整，最低 1 mB |
| 结算 | 默认每 20 Tick 合并结算一次，避免高倍率重复执行 |
| 输入 | 4 个可堆叠植物模板和时间流体；堆叠数量作为并行种植数量 |
| 堆叠流体 | 不超过最大堆叠数一半为 1 倍，超过一半为 2 倍 |
| 输出 | 基础 16 格分页产物槽，每张容量升级增加 16 格，最多 64 格 |
| 自动 I/O | 支持种子与时间流体输入，以及产物输出 |

容量、流体、超频、创造和时运升级可用于温室大棚。最多安装 3 个时运升级，每张对应 1 级原版时运；范围、过滤、降频和精准升级不支持。

## 植物兼容

- 内置小麦、胡萝卜、马铃薯、甜菜根、南瓜、西瓜、下界疣和可可豆配方。
- 通过公开 Crop Registry 自动支持所有已启用的 Mystical Agriculture 作物。
- Mystical Agradditions 作物注册在同一 Crop Registry 中，因此会自动兼容。
- 绑定到带 `age` 属性作物方块的其他模组种子会被通用识别。
- 通用支持采用 `BushBlock` 实现的花朵、树苗、蘑菇和同类模组植物；植物自身掉落表作为默认产物。
- 内置原版树苗配方会生成对应原木并返还树苗。模组树苗无法通过稳定公共 API 推导完整树形产物时，可由数据包定义多个明确产物。
- 收获使用成熟作物的真实掉落表，因此小麦会同时产出小麦与种子，模组作物也会保留自己的副产物和时运逻辑。
- 神秘农业作物的时间流体成本按作物等级平方增长。
- 整合包可以添加 `jdte:greenhouse` 配方扩展其他植物，并通过 `outputs` 和 `use_loot_table` 选择明确产物或真实掉落。

JEI 分类会显示植物模板、基础时间流体、基础 FE 和预览产物；点击温室界面的苗床进度区域也可以打开分类。

温室只缓存 4 个当前植物模板的定义，并让四条生产线共享单次批量上限、轮换资源优先级。每株每次结算最多进行 4 次真实掉落采样后批量放大，不会创建掉落物实体、扫描范围或按理论收获次数循环调用掉落表。

超频升级和创造升级都会将生产倍率固定为 64x。只有至少一条生产线具备所需资源和产物空间时，苗床进度才会推进。新收获会优先直接生成到相邻物品容器，并复用最近成功的方向；只有容器装不下的部分才进入温室内部输出槽。

横向摆放多个温室时，模型会自动连接：相邻侧的玻璃墙、立柱和顶部边框会消失，玻璃顶棚则延伸覆盖连接处。连接只改变外观，每台温室仍拥有独立库存、资源和升级。
