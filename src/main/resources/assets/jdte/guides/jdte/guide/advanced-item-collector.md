# 高级物品拾取器

<BlockImage id="jdte:advanced_item_collector" scale="2" />

高级物品拾取器使用 JDT 物品拾取器的模型和机器界面，并提供 8 个标准升级槽。

| 项目 | 行为 |
|------|------|
| 收集时机 | 掉落物入世前，并低频扫描已有掉落物 |
| 输出 | 机器朝向一侧的相邻容器 |
| 内部缓存 | 无 |
| 支持升级 | 范围、过滤 |
| 大堆叠保护 | 默认阈值 10,000,000，可配置 |

新掉落物仍使用事件驱动路径，不执行范围扫描：服务端只检查掉落物所在区块已索引的拾取器，并将同一 tick 内的相同物品聚合为一次插入。已有掉落物使用独立的有界轮询扫描：每个服务端 tick 最多扫描一台到期机器，默认每台至少间隔 10 tick，单次最多处理 256 个掉落实体；三个选项均可配置。

## 使用方法

1. 将高级物品拾取器朝向一个具有物品能力的容器。
2. 在界面中配置 X/Y/Z 范围、偏移、过滤和红石模式。
3. 范围内新产生和已经存在的掉落物都会写入相邻容器。
4. 容器空间不足时，只收集可插入的部分，剩余物品仍会正常生成。

玩家破坏范围内的容器时，如果某个槽位达到配置阈值（默认 10,000,000 件），拾取器会在生成掉落实体前通过物品能力直接转移该超大堆叠，从而避免一次创建数十万个物品实体。触发后的超大槽位必须完整转移；若目标容器或过滤器无法全部接收，本次破坏会被取消并显示提示。该功能默认开启，可在 `jdte.advancedItemCollector` 配置分类中关闭或调整阈值。

超大堆叠阈值只用于破坏容器前的安全保护。面对 AE2 ME 接口或 ExtendedAE 扩展/超大接口时，普通掉落物和超大堆叠都会先模拟接口普通物品能力；如果不能一次完整接收，拾取器才查询一次 `ME_STORAGE`，并在网络容量充足时用一次长整型操作原子直传。ME 网络也无法完整接收时会回退普通部分插入，剩余数量继续留在世界中。该逻辑与物品数量无关，可通过 `meDirectTransferEnabled` 关闭。

高级物品拾取器只接受范围升级和过滤升级；容量、流体、速度、创造及其他对事件驱动拾取没有实际作用的升级不能放入。八个升级槽使用 JDTE 的左右双列布局；机器本身没有内部物品缓存，也不生成物品流动粒子。
