# 实体抑制器

实体抑制器使用高级传感器式界面和八个升级槽，在配置范围内以事件驱动方式控制实体、方块实体动态渲染或粒子。

## 主要模式

- **抑制实体**：取消匹配实体的更新，使 AI、移动、投射物和实体计时停止。
- **禁止实体**：在实体生成检查或加入世界时阻止匹配实体出现。
- **禁用粒子**：在客户端创建粒子前拦截范围内的粒子。
- **禁用实体渲染**：跳过范围内匹配实体的客户端渲染，不影响实体更新、碰撞或服务端逻辑。
- **禁用方块实体渲染**：跳过范围内的动态方块实体渲染，兼容使用标准方块实体渲染器的模组；普通方块模型仍然显示。

实体模式可以选择敌对生物、被动生物、所有生物、指定实体类型、非生物实体或所有类型。过滤槽使用刷怪蛋指定实体类型，并可切换白名单或黑名单。粒子和方块实体渲染没有实体类型，目标与名单按钮在这两种模式下不可用。

玩家和载具关系中的实体始终受到保护。命名、已驯服和 Boss 实体默认受到保护，可在配置中调整。范围、过滤、容量和创造升级可用于此机器。

抑制实体模式会同时冻结服务端与客户端实体更新，并阻止被冻结的掉落物被拾取。禁止实体模式可通过默认关闭的 `removeExistingEntities` 配置，每 20 Tick 分散清除一次范围内已有的匹配实体。

渲染模式只减少客户端渲染开销，不会停止实体、方块实体或模组网络的服务端更新。禁用方块实体渲染会隐藏完整的动态渲染器，因此箱子、告示牌文字或完全依赖动态渲染器的机器可能显示不完整。实体抑制器自身始终保留渲染。
