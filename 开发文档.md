# JDT Extras 开发文档

## 项目概述

JDT Extras 是 Just Dire Things (JDT) 的扩展模组，为 JDT 机器添加升级系统和时间加速器。

- **模组ID**: `jdte`
- **版本**: 0.1.0
- **Minecraft版本**: 1.21.1
- **NeoForge版本**: 21.1.230+
- **JDT版本**: 1.5.7+

---

## 功能模块

### 1. 升级系统

#### 升级类型 (`UpgradeType`)

| 类型 | 序列化名 | 每台机器上限 | 效果 |
|------|----------|--------------|------|
| CAPACITY | `capacity` | 3 | FE和流体容量翻倍 |
| OVERCLOCK | `overclock` | 1 | 1 tick运行，每tick执行两次，3x能耗 |
| UNDERCLOCK | `underclock` | 1 | 40 tick运行，80%能耗降低 |
| FLUID | `fluid` | 3 | 流体容量翻倍 |
| FLUID_STORAGE | `fluid_storage` | 1 | Clicker专用：添加内部流体储罐 |
| GENERATOR | `generator` | 1 | 发电机专用：2x燃料→3x发电 |
| RANGE | `range` | 2 | 区域机器：半径上限翻倍 |
| FILTER | `filter` | 2 | 过滤机器：添加额外过滤槽 |

#### 升级处理器 (`UpgradeItemStackHandler`)

- 继承自 `ItemStackHandler`
- 默认4个槽位，扩展版本8个槽位
- 验证逻辑：检查升级类型与机器兼容性
- 容量同步：通过 `UpgradeHelper.syncCapacities()` 动态调整

#### 关键类

- `UpgradeHelper` - 升级辅助工具类
- `UpgradeSlot` - 升级槽位GUI组件
- `UpgradeItemStackHandler` - 升级物品处理器
- `ExtendedUpgradeItemStackHandler` - 扩展升级处理器（8槽）

### 2. 时间加速器

#### 基础时间加速器

- 仅消耗时间流体
- 默认4x加速，超频后16x
- 流体容量：1000 mB

#### 高级时间加速器

- 消耗时间流体 + FE能量
- 可调节倍率：1-128x
- 超频后256x
- FE容量：200,000

#### 加速原理

```java
// 每tick执行
for (BlockPos pos : area) {
    BlockEntityTicker ticker = blockState.getTicker();
    for (int i = 0; i < multiplier; i++) {
        ticker.tick(level, pos, state, blockEntity);
    }
}
```

#### 关键类

- `TimeAcceleratorBE` - 时间加速器基类
- `BasicTimeAcceleratorBE` - 初级时间加速器
- `AdvancedTimeAcceleratorBE` - 高级时间加速器
- `TimeAcceleratorEffectEntity` - 视觉效果实体

### 3. 扩展机器

扩展机器是JDT T2机器的升级版本，拥有8个升级槽。

| 方块 | 方块实体 | 继承自 |
|------|----------|--------|
| ExtendedClickerBlock | ExtendedClickerBE | ClickerT2BE |
| ExtendedBlockBreakerBlock | ExtendedBlockBreakerBE | BlockBreakerT1BE |
| ExtendedBlockPlacerBlock | ExtendedBlockPlacerBE | BlockPlacerT1BE |
| ExtendedBlockSwapperBlock | ExtendedBlockSwapperBE | BlockSwapperT1BE |
| ExtendedDropperBlock | ExtendedDropperBE | DropperT1BE |
| ExtendedSensorBlock | ExtendedSensorBE | SensorT1BE |
| ExtendedFluidCollectorBlock | ExtendedFluidCollectorBE | FluidCollectorT1BE |
| ExtendedFluidPlacerBlock | ExtendedFluidPlacerBE | FluidPlacerT1BE |

## Mixin 注入

### Mixin列表

| Mixin | 目标类 | 类型 | 功能 |
|-------|--------|------|------|
| BaseMachineBEMixin | BaseMachineBE | 服务端 | 同步升级容量 |
| BaseMachineBEPopupMixin | BaseMachineBE | 服务端 | 弹窗位置持久化 |
| BaseMachineBlockMixin | BaseMachineBlock | 服务端 | 超频双倍执行 |
| BaseMachineContainerMixin | BaseMachineContainer | 服务端 | 注入升级槽位 |
| BaseMachineContainerFilterMixin | BaseMachineContainer | 服务端 | 扩展过滤槽 |
| BaseMachineScreenMixin | BaseMachineScreen | 客户端 | 升级弹窗UI |
| ScreenMixin | Screen | 客户端 | 访问font字段 |
| AbstractContainerScreenMixin | AbstractContainerScreen | 客户端 | 访问imageWidth等字段 |
| FilterBasicHandlerAccessor | ItemStackHandler | 服务端 | 扩展过滤处理器容量 |
| SlotAccessor | Slot | 客户端 | 动态调整槽位位置 |
| EnergyStorageAccessor | EnergyStorage | 服务端 | 动态调整能量容量 |
| FluidTankAccessor | FluidTank | 服务端 | 动态调整流体容量 |

### Mixin最佳实践

1. **不要继承父类**：Mixin类不应显式继承任何类（除Object）
2. **使用@Shadow访问字段**：对于目标类的字段使用@Shadow
3. **使用反射调用父类方法**：当方法不在目标类中时使用反射
4. **避免每帧更新**：仅在状态变化时更新UI
5. **保存状态时机**：仅在mouseReleased时保存，而非每帧

## 设计规范

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 方块实体 | `Extended{Machine}BE` | `ExtendedClickerBE` |
| 方块 | `Extended{Machine}Block` | `ExtendedClickerBlock` |
| 容器 | `Extended{Machine}Container` | `ExtendedClickerContainer` |
| 升级类型 | `{Effect}_UPGRADE` | `CAPACITY_UPGRADE` |
| Mixin方法 | `jdte${methodName}` | `jdte$addUpgradeSlots` |
| Mixin字段 | `jdte${fieldName}` | `jdte$upgradePopupOpen` |

### 注册顺序

1. `JDTEBlocks` - 方块注册
2. `JDTEItems` - 物品注册
3. `JDTEBlockEntities` - 方块实体注册
4. `JDTEMenus` - 菜单注册
5. `JDTEEntities` - 实体注册
6. `JDTEAttachments` - 数据附件注册
7. `JDTECreativeTabs` - 创造模式标签页

### 能力注册

在 `JDTE.registerCapabilities()` 中注册：

```java
// 流体能力
event.registerBlock(Capabilities.FluidHandler.BLOCK, provider, blocks...);

// 能量能力
event.registerBlock(Capabilities.EnergyStorage.BLOCK, provider, blocks...);
```

## 执行步骤

### 添加新升级卡

1. 在 `UpgradeType` 枚举中添加新类型
2. 在 `JDTEItems` 中注册物品
3. 在 `UpgradeItemStackHandler.isItemValid()` 中添加验证逻辑
4. 在 `UpgradeHelper` 中添加辅助方法（如需要）
5. 在 `JDTECreativeTabs` 中添加到创造模式标签页
6. 创建模型文件 `models/item/{name}.json`
7. 添加翻译 `lang/en_us.json` 和 `lang/zh_cn.json`
8. 创建合成配方 `recipe/{name}.json`

### 添加新扩展机器

1. 创建方块实体类继承T1机器，实现 `ExtendedUpgradeMachine`
2. 创建方块类继承 `BaseMachineBlock`
3. 创建容器类继承T2容器，重写 `stillValid()`
4. 在 `JDTEBlocks`、`JDTEBlockEntities`、`JDTEMenus`、`JDTEItems` 中注册
5. 在 `JDTE.registerCapabilities()` 中注册能量/流体能力
6. 在 `JDTEClientSetup` 中注册屏幕和渲染器
7. 创建资源文件（blockstate、模型、战利品表、配方）
8. 添加翻译

### 添加新Mixin

1. 创建Mixin类，使用 `@Mixin(TargetClass.class)`
2. 使用 `@Shadow` 访问目标类字段
3. 使用 `@Inject`、`@ModifyConstant` 等注入方法
4. 使用 `@Unique` 定义私有辅助方法
5. 在 `mixins.jdte.json` 中注册（mixins或client列表）
6. 编译测试

## 调试技巧

### 启用调试日志

```java
private static final Logger LOGGER = LogUtils.getLogger();

// 在关键位置添加
LOGGER.info("[JDTE-DEBUG] message: {}", variable);
```

### Mixin调试

1. 检查 `mixins.jdte.json` 中的注册
2. 确认目标类和方法存在
3. 使用 `@Shadow` 时确保字段名正确
4. 使用反射时处理异常

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Mixin apply failed | @Shadow字段不存在 | 检查字段名，使用反射 |
| No refMap loaded | Mixin配置错误 | 检查mixins.jdte.json |
| InvalidAccessor | Accessor字段类型错误 | 使用javap检查实际类型 |
| 界面闪退 | Container的stillValid检查 | 确保检查正确的方块类型 |
